# OLA Ride Analytics — End to End Data Analysis Project
SQL | Power BI | Microsoft Excel

 Business Problem
OLA, one of India's largest ride-hailing platforms, needed 
in-depth analysis of ride booking data to understand operational 
performance, identify cancellation patterns, analyse revenue 
streams, and improve overall customer and driver experience. 
This project simulates a real-world data analyst role at OLA — 
answering critical business questions using SQL and presenting 
findings through an interactive Power BI dashboard.

Tools Used
- SQL (MySQL) — Data extraction and business question analysis
- Power BI — Interactive dashboard and KPI visualisation
- Microsoft Excel — Data validation and supplementary analysis
- DAX — Measures and calculated fields in Power BI

 Dataset
50,000+ OLA ride booking records for Bengaluru covering one 
month of operations with 20+ fields including:
- Booking ID, Status, Vehicle Type
- Pickup and Drop Locations
- Ride Distance and Duration
- Payment Method and Booking Value
- Customer and Driver Ratings
- Cancellation Reasons

SQL Analysis — Business Questions Answered
1. Retrieve all successful bookings
2. Find the average ride distance for each vehicle type
3. Get the total number of cancelled rides by customers
4. List the top 5 customers who booked the highest number of rides
5. Get the number of rides cancelled by drivers due to personal 
   and car related issues
6. Find the maximum and minimum driver ratings for Prime Sedan bookings
7. Retrieve all rides where payment was made using UPI
8. Find the average customer rating per vehicle type
9. Calculate the total booking value of rides completed successfully
10. List all incomplete rides along with the reason

 Power BI Dashboard — KPIs and Visuals
Key KPIs Tracked:
- Total Bookings and Booking Success Rate
- Total Revenue and Revenue by Payment Method
- Average Ride Distance by Vehicle Type
- Cancellation Rate — by Customer and by Driver
- Average Customer Rating and Driver Rating
- Ride Volume Over Time (Daily Trend)
- Top Customers by Booking Value
- Revenue Split by Vehicle Type

Dashboard Features:
- Dynamic slicers for Date, Vehicle Type, Booking Status, 
  and Payment Method
- Drill-through filters for customer-level and 
  driver-level analysis
- KPI cards with trend indicators
- Bar, line, donut charts and matrix tables

Key Insights
- Booking success rate of 62% — 38% of bookings were 
  either cancelled or incomplete
- Weekend ride demand significantly higher — peak bookings 
  on Friday and Saturday nights
- UPI was the most preferred payment method across all 
  vehicle types
- Driver cancellations concentrated in specific vehicle 
  categories — indicating driver allocation inefficiency
- Prime Sedan had the highest average driver rating 
  among all vehicle types

 Business Recommendations
- Address driver allocation gaps in high-cancellation 
  vehicle categories to improve success rate
- Introduce weekend surge incentives for drivers to 
  reduce cancellations during peak demand
- Focus retention efforts on top 5% of customers who 
  contribute disproportionately to total booking value
